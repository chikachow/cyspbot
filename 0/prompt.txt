# cyspbot deployment pipeline

This private repository deploys `chikachow/cyspbot` to production.

The source implementation is pinned as the `src/cyspbot` git submodule. This
repository owns the production Wrangler configuration overlays, Cloudflare
resource identifiers, route patterns, GitHub Actions deployment workflows, and
deployment runbook.

## Source

`src/cyspbot` pins the source commit that is eligible for production
deployment. Updating that submodule updates the source revision this repository
builds, checks, dry-runs, and deploys.

```bash
git submodule update --init --recursive
node --run source:install
node --run source:check
node --run deploy:dry-run
```

## Workers

cyspbot deploys the Workers listed in `scripts/deploy-configuration.mjs`.
Production routes, bindings, variables, and Cloudflare resource identifiers live
in `config/workers/*.wrangler.jsonc`.

Each Worker config filename must match the Worker name:
`config/workers/<worker-name>.wrangler.jsonc`.

The deployment script runs a strict Wrangler dry-run for every configured Worker
before a production deploy. `node --run deploy:dry-run` stops after the dry-runs.
`node --run deploy` runs the dry-runs first, then deploys every Worker with
Wrangler strict checks enabled.

## Cloudflare Configuration

The production Wrangler overlays bind existing Cloudflare resources. Keep those
values in `config/workers/*.wrangler.jsonc`, not duplicated in documentation.
Worker secret values are supplied by Cloudflare Secrets Store; do not add secret
values to the Wrangler overlays.

## Local Deployment

Authenticate Wrangler locally:

```bash
pnpm -C src/cyspbot exec wrangler login
```

For non-interactive deployments, provide the Cloudflare credentials consumed by
Wrangler. The deploy scripts run Node with `--env-file-if-exists=.env`, and
explicit environment variables take precedence over `.env` values.

```bash
node --run source:install
node --run deploy:dry-run
node --run deploy
```

## GitHub Actions

CI is orchestrated by `.github/workflows/ci.yml`:

- `.github/workflows/ci-source.yml` installs pinned source dependencies and runs
  `node --run source:check`.
- `.github/workflows/ci-deploy-dry-run.yml` installs the same pinned source
  dependencies and runs `node --run deploy:dry-run`.
- The aggregate `ci` job fails unless all required CI jobs pass.

Production deployment runs from `.github/workflows/deploy.yml` after a
successful `ci` workflow on `main`. The workflow owns its required secrets,
stale-main guard, deployment command, and post-deploy probes.
